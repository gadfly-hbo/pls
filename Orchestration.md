## 一、角色与职责

| 角色 | 名称 | 职责边界 |

| X | 总控 Agent | 产品架构、数据安全、画像标签体系、跨域接口契约、DB/schema 审批、任务拆卡、代码终审 |

## 三、域边界

### D · 数据画像

范围：

- 平台 DMP 聚合画像导入。
- 历史 SKU 销售与画像数据清洗。
- 商品基础属性字典。
- 渠道画像字段标准化。
- 脱敏、聚合、数据质量报告。

首批交付：

- 历史 SKU 宽表规范。
- DMP 标签映射规则。
- 数据安全检查清单。

### M · 模型预测

范围：

- 商品 DNA 特征工程。
- 商品图文解析策略。
- 新品画像预测模型。
- 商品画像与渠道画像匹配算法。
- 时间切分回测、Top 标签命中率、置信度。

首批交付：

- 商品画像预测 MVP 方案。
- 匹配度计算公式与评估指标。
- 反馈纠偏机制。

### A · 应用后端

范围：

- API 设计。
- 画像生成 pipeline。
- 任务状态与结果持久化。
- 权限、审计、异步任务。
- 与 D/M/V 的接口落地。

首批交付：

- P0 API 契约草案。
- 核心数据对象与存储草案。
- pipeline 状态机草案。

### V · 前端决策

范围：

- 新品画像预测工作台。
- 渠道人货匹配工作台。
- 匹配热力图。
- 画像详情、置信度、风险提示。
- 分货、投流、熔断建议展示。

首批交付：

- P0 信息架构。
- 两个工作台低保真流程。
- 决策输出字段清单。

## 四、接缝层规则

以下内容由 X 总控持有：

- 统一画像标签体系。
- 跨域共享类型。
- DB schema 和 migration。
- API 命名规范。
- 数据安全红线。
- 任务拆卡和验收标准。
- 最终代码终审。

域 Agent 如需修改接缝层，必须先回流总控。总控可在 brief 中写死口径后委派代笔，但终审不跳过。

## 五、任务生命周期

任务以 `docs/wiki.html` 的 `TASKS` 为派发真源。

状态：

- `todo`：待派发。
- `doing`：进行中。
- `done`：已完成并经总控终审。

流程：

1. 总控拆卡，写清目标、范围、红线、验收。
2. 域 Agent 复制任务 brief 开工。
3. 域 Agent 完成后回报改动、验证、风险。
4. 总控终审，通过后把任务改为 `done`。
5. 阶段结束时，总控把 done 任务归档到 changelog。

## 六、P0 启动顺序

```
第 0 步：X 冻结画像标签体系、数据红线、P0 契约
第 1 步：D/M/A/V 并行完成各自 P0-A 任务
第 2 步：X 集成评审，确定 API、数据表、工作台闭环
第 3 步：实现 MVP 骨架和 demo 数据闭环
第 4 步：端到端验收：新品输入 → 预测画像 → 匹配渠道 → 输出建议
```
