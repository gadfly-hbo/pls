推荐使用流程

小修小改

  1. Codex 直接读代码、改代码。
  2. 需要测试保障时用 matt-tdd。
  3. 完成后用 matt-code-review 或 agentops-review。
  4. 跑最小验证，Codex 汇总结论。

复杂功能

  1. 用 matt-grilling 或 matt-grill-with-docs 压测需求。
  2. 用 matt-to-prd 固化需求。
  3. 用 matt-to-issues 拆任务。
  4. Codex 主控实现；必要时派 OpenCode / MimoCode / Antigravity 做独立子任务。
  5. 每个辅 agent 必须交 handoff。
  6. Codex 集成、测试、最终说明。

Bug 修复

  1. 用 matt-diagnosing-bugs 先复现和定位。
  2. 明确根因后再改。
  3. 关键逻辑用 matt-tdd 补测试。
  4. 用 matt-code-review 检查回归风险。

UI 改造

  1. 用 ui 或 product-ui-redesign。
  2. 先截图审计，不先改颜色。
  3. 写 3-5 条产品 UI 原则。
  4. 定义组件系统或页面切片。
  5. 实现一页或一个 workflow。
  6. 用 Playwright / browser 截图验证桌面和移动端。
  7. 用 ui-polish-pass 做最后一轮细节。
