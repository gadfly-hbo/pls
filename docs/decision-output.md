# PLS P0 运营建议与决策输出设计

> 归属：V 前端决策
> 状态：P0 设计
> 对应任务：V-P0-4

## 1. 目的
本文件定义 P0 阶段针对商品与渠道匹配结果（MatchResult）在前端的具体展示形态与决策输出字段。系统不能仅抛出干冷的“分数”，而是要翻译成运营可直接理解与执行的建议动作。

## 2. 建议类型与对应行动 (Recommendations)

后端 API (`MatchResult.recommendation`) 返回枚举值，前端需要进行二次包装并高亮渲染：

| 推荐枚举 (后端) | 视觉呈现 (前端) | 决策话术与建议动作 | 适用场景 |
|---|---|---|---|
| `priority_launch` | 🔴 **高亮标签 (红/橙)** / 重点推 | **“重点铺货 / 强投流”**<br>该商品高度契合渠道核心客群，建议倾斜资源放大打爆。 | 高匹配分 (>=0.7)，且置信度高 |
| `test_launch` | 🟢 **常态标签 (绿)** / 试水 | **“小批次铺货 / 小额测试”**<br>人群有交集但不绝对，建议小批量上架并用达人内容或小额投流跑量测试。 | 中等匹配分 (0.5-0.7) |
| `observe` | 🟡 **警戒标签 (黄)** / 观望 | **“暂缓分货 / 自然流量观察”**<br>匹配特征不清晰，不建议盲目配货或投流，可先上架观察自然转化。 | 匹配分偏低，或预测置信度低 |
| `avoid` | ⚫ **警示标签 (灰/黑)** / 熔断 | **“熔断拦截 / 避免铺货”**<br>商品 DNA 与该渠道客群存在严重冲突（如价格带越级、风格严重不符），建议取消分货。 | 匹配分极低，或出现严重负向阻碍 |

## 3. 抽屉 / 弹窗展示核心字段清单

当用户在“热力图”点击某一个商品 × 渠道的单元格时，弹出的抽屉需展示以下结构化信息：

### 3.1 结论概览区
- **匹配度评分 (Match Score)**：如 `71 分`（环形进度条或显著数值展现）。
- **置信度 (Confidence)**：提示模型判断的确定性（如 `66%`），若低于 50% 需要旁置警告 Icon 增加预警。
- **核心结论 (Recommendation)**：基于上一节的建议类型，大字号输出相应的核心话术动作。

### 3.2 归因解释区 (Drivers)
- **正向匹配点 (Positive Drivers)**：
  - 核心特征对比：商品特征（如：`style.minimal`, `price.mid`）与 渠道画像中人群强需求点的重叠项。
  - 解释说明：“该渠道【25-34岁简约通勤女性】占比较高，与新品核心 DNA 高度吻合。”
- **负向拦截点 (Negative Drivers)**：
  - 展示引发低分或熔断的冲突标签。
  - 解释说明：“该渠道对【促销敏感/低价】高度敏感，但当前商品【价格带：高客单】超出核心受众心理预期。”

### 3.3 风险与提示区 (Risks & Flags)
前端拦截 `MatchResult.risks` 和 `qualityFlags` 并统一转化为人类可读警告，渲染为底色警示框：
- `channel_price_sensitivity_gap` -> ⚠️ “渠道价格敏感度不匹配：商品价格可能超出渠道主流用户承受范围。”
- `channel_sample_thin` -> ⚠️ “渠道样本量不足：当前该渠道导入的历史画像数据较薄弱，评级结果仅供参考。”
- `prediction_below_threshold` -> ⚠️ “新品特征不清晰：商品录入 DNA 特征稀疏，导致源头的人群画像预测置信度偏低。”
- `no_common_tags` -> ⚠️ “无显著共性标签：新品画像与渠道主流画像未发现交集特征。”

## 4. 开放问题决策响应
- **是否需要导出 Excel/CSV？**
  - **P0 决策**：虽然 P0 以打通主干流程闭环为主，但基于运营实际痛点，在人货匹配热力图页面右上角，预留 **“导出匹配报告 (CSV)”** 按钮。一期前端可直接拉取当前热力图和 `MatchResult` 派生结果，在浏览器端合成纯文本 CSV 文件。
  - **导出口径**：CSV 默认包含 `skuId`、`channelId`、`matchScore`、`matchConfidence`、`recommendation`、driver tagId、`risks`、`generatedAt` 等决策结果字段；用户授权导入或要求产品化的业务字段也可按页面场景导出。
- **P0 阶段的交互原型深度**：
  - **P0 决策**：直接进入轻量工程实现或输出低保真线框图。无须精细的高级动效，采用基础组件库（如类似 Ant Design 的中后台设计语言）快速堆叠出具备表单录入、结果消费和热力图映射能力的工作台，确保接口和流程可走通。
