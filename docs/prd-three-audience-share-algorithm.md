# 森马三大人群占比算法基础 PRD

## 目的

在 PLS 模型域建立可复算、可追溯的森马三大人群占比算法基础能力，把七类渠道的原生人群分布映射为：

- A 类「质感流行派」
- B 类「都市体面家」
- C 类「百搭优选客」

本轮只交付对象无关的算法函数和 contract test，不接入渠道画像数据结构、数据库、Server API 或 Frontend。

## 范围

### 本轮交付

1. 支持抖音、天猫、京东、线下、唯品会、视频号、拼多多七类渠道。
2. 只接受渠道指定的份额层原生人群分布。
3. 默认对已覆盖部分归一化，使 A+B+C=100%。
4. 支持可选专家先验输入，用于未覆盖部分的临时重算。
5. 输出 coverage、计算模式、算法版本和质量标记。
6. 京东使用 v2.1.0 校准矩阵；十大靶群每行 `A+B+C=1`，合法十大靶群输入 `coverage=1`。
7. 使用天猫六大行业特色人群样例完成可复算 contract test。

### 非目标

1. 不定义渠道、店铺或账号画像的数据结构。
2. 不修改 `AudienceProfile`、DB schema、migration 或导入包。
3. 不新增 Server API。
4. 不修改 Frontend。
5. 不持久化算法结果或专家先验。
6. 不从年龄、城市、消费力、兴趣、折扣敏感度等行为或基础属性推断份额。
7. 不实现跨期趋势、行为强度、动态 `±npp` 或完整置信度模型。
8. 不提供跨渠道 mapping fallback。

## 使用方式

调用方提供：

- 森马品牌范围。
- 七类受支持渠道之一。
- 该渠道指定原生人群体系中的标签及 `0-1` 占比。
- 可选的 A/B/C 专家先验，三项合计必须为 1。

算法返回三类占比、coverage、计算模式、版本、未映射标签和质量标记。完整类型与矩阵见 `docs/model-three-audience-share-contract.md`。

## 示例

天猫样例的六大行业特色人群原始占比合计为 `0.9005`。默认模式忽略未覆盖的 `0.0995`，对已覆盖贡献归一化，预期结果为：

| 人群 | 预期占比 |
|---|---:|
| A 类「质感流行派」 | 约 46.99% |
| B 类「都市体面家」 | 约 24.07% |
| C 类「百搭优选客」 | 约 28.93% |

允许因浮点运算和最终舍入出现 `0.01pp` 差异；算法内部不得先舍入再计算。

## 验收标准

1. 七类渠道固定矩阵均有正向 contract test。
2. 京东使用业务确认目标校准矩阵；年均口径对齐 2025 年 `22.1/32.9/45.0`、2026 年归一化后的 `21.12/34.73/44.14`，2024 年 `22.5/32.6/44.8` 因缺少原始十大靶群文件仅记录为目标。
3. 默认模式只归一化已覆盖部分，并显式输出 coverage；京东校准后在十大靶群合计为 1 时 coverage 为 1。
4. 专家先验模式严格执行 coverage 软回填，不持久化输入。
5. 未知体系、空覆盖、非法 share、非法 prior 均显式失败或返回不可计算，不静默 fallback。
6. 天猫确认样例结果满足本 PRD 的预期值。
7. `apps/model` typecheck 与新增 contract test 通过。

## 注意事项

- 两份来源文档的京东正文公式与映射表 v2.0.2 均保留 uncovered 口径；v2.1.0 京东校准矩阵以 `docs/model-jd-three-audience-calibration.md` 为准。
- 来源文档没有提供可执行的动态 `±npp` 公式，因此本轮不得伪造误差区间。
- 来源文档没有给出各渠道专家先验值；算法只接受调用方显式传入的 prior。
- 后续渠道画像接入必须另行冻结真实结构、字段来源、对象身份、时间窗口和导入契约。
