> 依据：`docs/channel-profile-2.0-plan.md`、`docs/api-contract.md`、`docs/pipeline-design.md`、`docs/p3-db-schema-migration-contract.md`、`docs/tools-module-design.md` 及现有真实路由代码。

## 1. 任务范围与边界

### 1.1 目标（必须实现）
1. 新增 5 张业务表 + 5 个 latest 视图：
   - `channel_object`（合并 ChannelEntity / MarketingEvent / BusinessScenario 的 append-only 版本表）
   - `channel_object_binding`（绑定关系）
   - `audience_profile`（人群画像）
   - `product_fit_profile`（商品适配画像）
   - `channel_entity_view`（仅长期渠道实体的 latest 投影，兼容 `/channels/entities`）
2. 新增 `channel-profile-object-library` 导入适配器：dry-run → quality report → confirm import → audit。
3. 新增 API 路由：
   - `GET /api/v0/channel-objects`（对象库列表）
   - `GET /api/v0/channel-objects/:canonicalObjectKey`（对象详情）
   - `GET /api/v0/channel-objects/:canonicalObjectKey/audience-profiles`（latest 人群画像）
   - `GET /api/v0/channel-objects/:canonicalObjectKey/product-fit-profiles`（latest 商品适配）
   - `GET /api/v0/channel-objects/:canonicalObjectKey/bindings`（绑定关系）
   - `POST /api/v0/admin/database/import-jobs/dry-run`（已有）扩展支持 `packageType: channel-profile-object-library`
   - `POST /api/v0/admin/database/import-jobs`（已有）扩展支持该 package 类型
4. 保持现有 `/channels` 与 `/channels/entities` 只读兼容，不破坏现有路径。
5. 导入必须走 `X-PLS-Admin-Token` / `Idempotency-Key` / `confirmText` / `db_admin_audit` 纪律。

### 1.2 非目标（不做）
- 不自动合并疑似重复对象。
- 不做完整 CRUD（对象库本体只支持轻量元数据编辑，不做批量删除、字段全量替换或版本回滚）。
- 不做版本回滚（版本删除沿用 `/admin/database/versions/:dataVersion`，但本期对象表暂不加入 `VERSIONED_TABLES`）。
- 不做流量/转化/运营约束画像。
- 不做 UI（V-P6-CHANNEL-4 负责）。
- 不修改 M 域匹配算法（M-P6-CHANNEL-2 负责）。

## 2. 设计决策

### 2.1 单表 vs 多表对象
采用**单表 `channel_object` + 类型列**承载 `ChannelEntity` / `MarketingEvent` / `BusinessScenario`，原因：
- 三类对象共享相同身份元数据（`canonicalObjectKey`, `objectVersionId`, `sourceBatchId`, `dataVersion`, `generatedAt`, `timeWindow`, `qualityFlags`）。
- 通过 `object_type` 列区分，查询时按类型过滤；`ChannelEntity` 独有的店铺/账号/地理字段放入 JSON 列 `attributes`。
- 活动和场景不是渠道实体，但共享对象库身份模型，单表便于统一版本控制与 latest 视图。

### 2.2 画像与对象分离
`audience_profile` 和 `product_fit_profile` 单独成表，通过 `canonical_object_key` 关联；原因是：
- 一个对象可有多版本画像，latest 视图按 `(canonical_object_key, profile_stage)` 取最新。
- 画像与对象独立生命周期，支持未来多来源画像（如 user_imported + derived_from_performance）。

### 2.3 兼容现有 channel_entity
- 不删除或修改现有 `channel_entity` 表和 `channel_entity_latest` 视图。
- 新对象库通过 `channel_object` 表提供 P6 语义；`channel_entity_view`（新视图）仅投影 `object_type` 为平台/商圈/店铺/账号的 latest 行，作为未来可选升级路径，但 `/channels/entities` 仍读取旧投影表。
- 本次 import 的 `ChannelEntity` 行**不**反向同步到旧 `channel_entity` 表，避免破坏 P2 投影语义。

### 2.4 导入幂等与 confirmText
- `confirmText = IMPORT CHANNEL OBJECT LIBRARY <sourceBatchId>`，与 `run_manifest.json.importAdapter.confirmText` 对齐。
- 正式导入写入 `data_import_job` + `batch` + `db_admin_audit`。
- 重复 sourceBatchId + dataVersion 通过 `INSERT OR REPLACE` 在对象表幂等，但 import 任务仍新建 `job_id`；若需严格幂等，通过 `Idempotency-Key` 复用响应。

### 2.5 数据包来源
复用 D-P6-CHANNEL-1 已冻结的样例包：`data/templates/channel-profile-object-library/sample_package/`。
- 读取 `channel_objects.jsonl` / `bindings.jsonl` / `audience_profiles.jsonl` / `product_fit_profiles.jsonl` / `quality_report.json`。
- 基础模板（basic_templates.csv）本期仅作为校验参考，不通过基础模板导入；只支持高级对象包导入。

### 3.5 Latest views

```sql
CREATE VIEW IF NOT EXISTS channel_object_latest AS
SELECT * FROM (
  SELECT *, ROW_NUMBER() OVER (PARTITION BY workspace_id, canonical_object_key ORDER BY generated_at DESC, rowid DESC) AS _latest_rank
  FROM channel_object
) WHERE _latest_rank = 1;

CREATE VIEW IF NOT EXISTS channel_object_binding_latest AS
SELECT * FROM (
  SELECT *, ROW_NUMBER() OVER (PARTITION BY workspace_id, binding_id ORDER BY generated_at DESC, rowid DESC) AS _latest_rank
  FROM channel_object_binding
) WHERE _latest_rank = 1;

CREATE VIEW IF NOT EXISTS audience_profile_latest AS
SELECT * FROM (
  SELECT *, ROW_NUMBER() OVER (PARTITION BY workspace_id, canonical_object_key, profile_stage ORDER BY generated_at DESC, rowid DESC) AS _latest_rank
  FROM audience_profile
) WHERE _latest_rank = 1;

CREATE VIEW IF NOT EXISTS product_fit_profile_latest AS
SELECT * FROM (
  SELECT *, ROW_NUMBER() OVER (PARTITION BY workspace_id, canonical_object_key ORDER BY generated_at DESC, rowid DESC) AS _latest_rank
  FROM product_fit_profile
) WHERE _latest_rank = 1;
```

## 4. API 设计

### 4.1 对象库路由
挂载在 `api.route("/channel-objects", channelObjects)`，顺序在 `/channels/entities` 与 `/channels` 之后，避免参数冲突。

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/channel-objects` | 列表；查询参数 `objectType`, `platformType`, `sourceBatchId`, `dataVersion`, `pageSize`（默认 50，最大 200）；默认查 `channel_object_latest`。 |
| GET | `/channel-objects/:canonicalObjectKey` | 详情；默认 latest，可传 `dataVersion` 查历史。 |
| GET | `/channel-objects/:canonicalObjectKey/audience-profiles` | 该对象的 latest 人群画像；可传 `dataVersion`。 |
| GET | `/channel-objects/:canonicalObjectKey/product-fit-profiles` | 该对象的 latest 商品适配画像；可传 `dataVersion`。 |
| GET | `/channel-objects/:canonicalObjectKey/bindings` | 与该对象相关的 binding（from 或 to）；可传 `bindingType`。 |
| POST | `/channel-objects/:canonicalObjectKey/bindings` | 轻量新增绑定（手动维护），本期预留实现或返回 501。 |

响应对象名使用 camelCase，例如 `canonicalObjectKey`, `objectVersionId`, `entityAttributes`, `duplicateCandidateKeys`, `manualReviewStatus`。

### 4.2 Admin 导入扩展
在 `admin-database.ts` 的 `POST /import-jobs/dry-run` 和 `POST /import-jobs` 中：
- `getPackageConfig('channel-profile-object-library')` 返回新配置。
- `import-packages.ts` 的 `PACKAGES` 增加条目，指向 `data/templates/channel-profile-object-library/sample_package`。
- `dryRun` 和 `executeImport` 分支增加 `channel-profile-object-library` 处理。

### 4.3 轻量编辑接口（P6 第一期）

为对象库 UI 提供最小写入面，只支持不触发版本变更的元数据补充，不覆盖导入生成的版本行：

| 方法 | 路径 | 说明 |
|---|---|---|
| PATCH | `/channel-objects/:canonicalObjectKey` | 轻量更新 `displayName`, `platformName`, `platformType`, `entityStatus` 或 `entityAttributes` 内的部分字段（如商圈半径、活动 `customTags`、场景说明等）。写入时生成新的 `object_version_id`（追加新版本），并更新 `manualReviewStatus`。 |
| PATCH | `/channel-objects/:canonicalObjectKey/bindings/:bindingId` | 修改绑定关系的元数据或 `qualityFlags`。 |
| POST | `/channel-objects/:canonicalObjectKey/bindings` | 手动新增绑定（from/to 对象必须在当前 workspace 存在）。 |
| DELETE | `/channel-objects/:canonicalObjectKey/bindings/:bindingId` | 软删除绑定（设置 `entityStatus = inactive`），不物理删除。 |

**PATCH 示例：**

```http
PATCH /api/v0/channel-objects/trade_area:mock_trade_area_city_walk HTTP/1.1
X-PLS-Workspace: ws_demo
Authorization: Bearer pls-p0-demo-token
Content-Type: application/json

{
  "displayName": "Mock City Walk Trade Area (Updated)",
  "entityAttributes": {
    "radiusKm": 4,
    "radiusSource": "user_config"
  },
  "manualReviewStatus": "confirmed_distinct"
}
```

响应返回新的 `objectVersionId` 与 `dataVersion`（追加为最新版本）。

## 5. Import 适配器实现

### 5.1 文件结构
新增 `apps/server/src/lib/import-channel-object-library.ts`：
- `dryRunChannelObjectLibrary(packageDir)`：读取样例包 JSONL，验证对象类型、父级引用、画像 lineage、重复标记，返回 `DryRunResult`。
- `executeChannelObjectLibrary(db, workspaceId, packageDir)`：将数据写入 `channel_object`, `channel_object_binding`, `audience_profile`, `product_fit_profile`。
- 统一写入 `data_import_job` + `batch` + `db_admin_audit`。

### 5.2 校验规则
- 对象类型必须是六类之一；否则 `invalid_object_type` 阻断。
- `marketing_event` / `business_scenario` 的 `targetObject` 必须是 `MarketingEvent` / `BusinessScenario`，否则 `event_or_scenario_as_channel_entity` 阻断。
- `store` / `account` 的 `parentCanonicalObjectKey` 若存在，必须能在本包或目标库找到（本期放宽：仅标记 `missing_parent_reference` warning，不阻断，因为业务包可能分多批导入）。
- 画像 `tagId` 必须命中 taxonomy（通过 `apps/server/src/lib/taxonomy.ts` 的 whitelist），否则移入 `unmapped_fields` 并标记 `unapproved_tag_id`。
- `possible_duplicate` 只生成 warning，不阻断。
- `generated_key_needs_review` 只生成 warning。
- `manual_entity_without_profile` 只生成 warning（渠道实体无 audience/profile）。

### 5.3 幂等写入
- 对象表按 `(workspace_id, canonical_object_key, data_version)` 主键 `INSERT OR REPLACE`。
- binding 表按 `(workspace_id, binding_id, data_version)` 主键幂等。
- 画像表按 `(workspace_id, profile_id, data_version)` 主键幂等。

## 9. 风险与回滚

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| 现有 `channel_entity` 表被误改 | 破坏 P2 实体读取 | 明确不修改 `CHANNEL_ENTITY_DDL` 和 `channel-entities.ts` 读取逻辑。 |
| `schema:check` 报 missing/extra | 迁移失败 | 新表加入 DDL 常量 + 版本化迁移 + `dangerous-ops.ts` / `admin-database.ts` 白名单同步更新。 |
| 样例包字段与真实导入不匹配 | import 失败 | 严格按 D-P6-CHANNEL-1 的 `channel_objects.jsonl` 字段映射；对未知字段放入 `raw` JSON。 |
| 导入污染 `ws_demo` | 数据损坏 | smoke 必须走临时 workspace；正式 import 只在用户指定的 workspace 上执行。 |
| 路由顺序冲突 | `/channel-objects/:key` 吞掉其他路径 | 在 `index.ts` 中把 `/channel-objects` 放在 `/channels` 和 `/channels/entities` 之后。 |
| 大 JSONL 加载内存问题 | 服务 OOM | 样例包很小；生产大文件需流式处理，本期不加。 |
